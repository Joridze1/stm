## Обзор

`stm32f401_test` — проект встроенного ПО для микроконтроллера `STM32F401CCU6`,
реализующий мини-игру в жанре аркадного шутера на OLED-дисплее `SSD1306 128x64`.

Прошивка:

- считывает положение джойстика по `ADC1`
- обрабатывает пользовательский ввод через `GPIO`
- формирует изображение в локальном кадровом буфере
- передаёт на дисплей только изменившиеся данные по `I2C1`
- поддерживает игровой цикл с обновлением состояния, отрисовкой и управлением скоростью

Репозиторий подходит как база для экспериментов с STM32, так и как компактный пример
организации прикладной логики поверх стандартного HAL-стека.

## Функциональность

- вывод графики на `SSD1306` без внешней графической библиотеки
- частичное обновление экрана вместо полной перерисовки на каждом кадре
- управление кораблём по оси X с чувствительностью, зависящей от отклонения джойстика
- выстрел и рестарт на одном входе `PA5`
- счётчик очков и экран завершения игры
- несколько активных противников со случайным респавном
- ограничение частоты игрового цикла

## Аппаратная конфигурация

| Узел | Пин / интерфейс | Назначение |
| --- | --- | --- |
| MCU | `STM32F401CCU6` | Основной контроллер |
| OLED | `PB8 = I2C1_SCL`, `PB9 = I2C1_SDA` | Дисплей `SSD1306 128x64` |
| Джойстик, ось X | `PA4 = ADC1_IN4` | Аналоговое управление по горизонтали |
| Кнопка | `PA5` | Выстрел в игре, рестарт после `GAME OVER` |

### Управление

| Действие | Способ |
| --- | --- |
| Движение влево / вправо | Джойстик на `PA4` |
| Выстрел | Нажатие `PA5` во время игры |
| Рестарт | Нажатие `PA5` после завершения игры |

## Архитектура

Логика проекта сосредоточена в [`Core/Src/main.c`](./Core/Src/main.c) и разделена на
три базовых этапа:

1. `Game_ProcessInput()` — чтение джойстика и кнопки.
2. `Game_Update()` — обновление состояния игрока, пули, противников и счёта.
3. `Game_Render()` — подготовка кадра и отправка изменений на дисплей.

Подход с буфером кадра даёт два практических преимущества:

- уменьшает мерцание OLED
- позволяет рисовать в пиксельных координатах, не ограничиваясь страницами SSD1306

### Ключевые модули и файлы

- [`Core/Src/main.c`](./Core/Src/main.c) — игровая логика, рендеринг, обработка ввода
- [`Core/Inc/main.h`](./Core/Inc/main.h) — базовые объявления проекта
- [`stm32f401_test.ioc`](./stm32f401_test.ioc) — конфигурация STM32CubeMX
- [`STM32F401CCUX_FLASH.ld`](./STM32F401CCUX_FLASH.ld) — скрипт линковки

## Сборка и прошивка

### Требования

- `STM32CubeIDE 1.15.x`
- установленный набор инструментов `GNU Tools for STM32`
- программатор/отладчик `ST-LINK`

### Сборка в STM32CubeIDE

1. Открыть `STM32CubeIDE`.
2. Импортировать репозиторий как существующий STM32CubeIDE-проект.
3. При необходимости открыть [`stm32f401_test.ioc`](./stm32f401_test.ioc) для проверки или регенерации конфигурации CubeMX.
4. Выбрать конфигурацию `Debug`.
5. Выполнить сборку проекта.
6. Прошить целевое устройство через `ST-LINK`.



## Параметры исполнения

- тактирование ядра: `HSI 16 MHz`
- интерфейс дисплея: `I2C1`, `100 kHz`
- вход джойстика: `ADC1_IN4`
- целевой шаг игрового цикла: `25 ms`
- расчётная частота кадров: около `40 FPS`
